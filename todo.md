## 🐛 Критические проблемы

### 1. IPv6 Warning показывается только один раз
**Файл:** `windows.py`
**Статус:** ✅ Решено — предупреждение показывается один раз за сессию (маркер `.ipv6_warned`).
**Примечание:** Если пользователь не отключил IPv6 после первого предупреждения, проблема остаётся на его стороне.

### 2. Отсутствие обработки ошибок при bind на порт
**Файл:** `windows.py`, `_run_proxy_thread()`
**Статус:** ✅ Решено — добавлена функция `_check_port_available()` с предварительной проверкой.

### 3. macOS Tray-приложение (pystray)
**Файл:** `macos.py`
**Статус:** ✅ Решено (v2.5.0) — переход на нативный macOS menu bar через `rumps`.
**Проблема:** pystray имеет ограничения на macOS, особенно в frozen-приложениях.
**Решение:** Полная переписка `macos.py` с использованием `rumps` для нативной интеграции.

### 4. MTProto proxy не пересылает данные (v2.1.0-v2.5.0)
**Файл:** `proxy/mtproto_proxy.py`, `_forward_data()`
**Статус:** ✅ Решено — добавлено подключение к серверу Telegram и двусторонняя пересылка.
**Проблема:** Функция `_forward_data()` принимала данные от клиента, но не пересылала их на сервер Telegram.
**Решение:** Реализовано подключение к `149.154.167.220:443` (DC2) и полноценная пересылка трафика в обе стороны.

### 5. MTProto proxy: захардкоженный DC2 (v2.5.0)
**Файл:** `proxy/mtproto_proxy.py`, `proxy/constants.py`
**Статус:** ✅ Решено — добавлена поддержка выбора DC через `--dc-id`.
**Проблема:** MTProto proxy использовал захардкоженный DC2 для подключения к Telegram.
**Решение:**
- Добавлены константы `DC_IP_MAP` и `DEFAULT_DC_ID` в `constants.py`
- Добавлен параметр `dc_id` в класс `MTProtoProxy` и функцию `run_mtproto_proxy()`
- Добавлен CLI аргумент `--dc-id` (по умолчанию = 2)
- Логирование подключения с указанием DC: `Connected to Telegram server DC2 ...`

### 6. MTProto proxy: утечка соединений при ошибке (v2.5.0)
**Файл:** `proxy/mtproto_proxy.py`, `_forward_data()`
**Статус:** ✅ Решено — закрытие клиентского соединения при ошибке подключения к Telegram.
**Проблема:** При ошибке подключения к серверу Telegram клиентское соединение оставалось открытым.
**Решение:** Добавлено закрытие `client_writer` с отправкой пустого ответа.

### 7. MTProto proxy: отсутствие валидации dc_id
**Файл:** `proxy/mtproto_proxy.py`, `proxy/mtproto_config.py`
**Статус:** ✅ Решено — добавлена валидация диапазона 1-5.
**Проблема:** Не было проверки корректности `dc_id`.
**Решение:**
- Валидация в `MTProtoProxy.__init__()` (ValueError при некорректном dc_id)
- Валидация в `run_mtproto_proxy()` (выход с ошибкой)
- Добавлено поле `dc_id: int = 2` в `MTProtoConfig`
- Сохранение/загрузка `dc_id` в JSON конфиг

### 8. MTProto proxy: общая обработка ошибок (v2.5.0)
**Файл:** `proxy/mtproto_proxy.py`
**Статус:** ✅ Решено — улучшена обработка исключений.
**Проблема:** Использовался общий `except Exception` без детализации.
**Решение:**
- `except asyncio.TimeoutError` — таймаут подключения к Telegram
- `except (ConnectionRefusedError, OSError)` — недоступность сервера
- `except ValueError` — некорректная расшифровка секрета
- Более точные сообщения об ошибках
