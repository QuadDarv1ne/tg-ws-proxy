# TG WS Proxy

**Автор:** Dupley Maxim Igorevich  
**© 2026 Dupley Maxim Igorevich. Все права защищены.**

Локальный SOCKS5-прокси для Telegram Desktop, который перенаправляет трафик через WebSocket-соединения к указанным серверам, помогая частично ускорить работу Telegram.
  
**Ожидаемый результат аналогичен прокидыванию hosts для Web Telegram**: ускорение загрузки и скачивания файлов, загрузки сообщений и части медиа.

## Как это работает

```
Telegram Desktop → SOCKS5 (127.0.0.1:1080) → TG WS Proxy → WSS (kws*.web.telegram.org) → Telegram DC
```

1. Приложение поднимает локальный SOCKS5-прокси на `127.0.0.1:1080`
2. Перехватывает подключения к IP-адресам Telegram
3. Извлекает DC ID из MTProto obfuscation init-пакета
4. Устанавливает WebSocket (TLS) соединение к соответствующему DC через домены `kws{N}.web.telegram.org`
5. Если WS недоступен (302 redirect) — автоматически переключается на прямое TCP-соединение

## 🚀 Быстрый старт

### Windows

#### Вариант 1: Готовый .exe (рекомендуется)

1. Перейдите на [страницу релизов](https://github.com/Flowseal/tg-ws-proxy/releases)
2. Скачайте **`TgWsProxy.exe`** (собирается автоматически через [Github Actions](https://github.com/Flowseal/tg-ws-proxy/actions))
3. Запустите файл
4. При первом запуске откроется окно с инструкцией по подключению Telegram Desktop
5. Приложение сворачивается в системный трей

> **⚠️ Важно:** Windows Defender может ошибочно помечать приложение как Wacatac (ложное срабатывание).  
> Решение: добавьте приложение в исключения или скачайте версию win7.

**Меню трея:**
- **Открыть в Telegram** — автоматически настроить прокси через `tg://socks` ссылку
- **Перезапустить прокси** — перезапуск без выхода из приложения
- **Настройки...** — GUI-редактор конфигурации (Ctrl+R — сохранить, Ctrl+Q — отмена)
- **Статистика** — просмотр статистики подключений
- **Открыть логи** — открыть файл логов
- **Выход** — остановить прокси и закрыть приложение

**Горячие клавиши:**
- `Ctrl+R` — сохранить настройки и перезапустить прокси
- `Ctrl+Q` — отменить и закрыть окно настроек
- `Escape` — закрыть окно

|---|---|---|

```bash

# С подробным логированием
python proxy/tg_ws_proxy.py -v
```

## Настройка Telegram Desktop

### Автоматически

ПКМ по иконке в трее → **«Открыть в Telegram»**

### Вручную

1. Telegram → **Настройки** → **Продвинутые настройки** → **Тип подключения** → **Прокси**
2. Добавить прокси:
   - **Тип:** SOCKS5
   - **Сервер:** `127.0.0.1`
   - **Порт:** `1080`
   - **Логин/Пароль:** оставить пустыми

## 🔧 Профилирование производительности

Проект включает инструменты для профилирования производительности.

### Использование профилировщика

```python
from proxy.profiler import (
    PerformanceProfiler,
    AsyncPerformanceProfiler,
    start_profiling,
    stop_profiling,
    print_profiling_stats
)

# Вариант 1: Профилирование отдельной функции
profiler = PerformanceProfiler()

# Синхронная функция
result = profiler.profile(my_function, arg1, arg2)
stats = profiler.get_stats()
profiler.print_stats()

# Асинхронная функция
async_profiler = AsyncPerformanceProfiler()
result = await async_profiler.profile_async(my_async_function, arg1)
async_profiler.print_stats()

# Вариант 2: Глобальное профилирование
start_profiling()

# ... выполнение кода ...

stop_profiling()  # Автоматически выведет статистику
```

### Интерпретация результатов

Профилировщик выводит:
- **Calls** — количество вызовов функции
- **Total(s)** — общее время выполнения (секунды)
- **Avg(ms)** — среднее время на вызов (миллисекунды)
- **Min(ms)** — минимальное время выполнения
- **Max(ms)** — максимальное время выполнения

### Оптимизация на основе профилирования

1. Найдите функции с наибольшим `Total(s)`
2. Проверьте `Avg(ms)` для выявления медленных операций
3. Обратите внимание на функции с большим `Calls`
4. Оптимизируйте "узкие места" в порядке убывания вклада
