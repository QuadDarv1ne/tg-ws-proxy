## 📱 Пошаговая инструкция подключения

### Шаг 1: Проверьте, что телефон в той же сети

**Важно:** Телефон и компьютер должны быть подключены к **одной Wi-Fi сети**!

**Как проверить:**
1. Откройте настройки Wi-Fi на телефоне
2. Посмотрите имя сети (например, "Home WiFi")
3. Сравните с именем сети на компьютере

### Шаг 2: Откройте браузер на телефоне

**Используйте:**
- ✅ Chrome (Android)
- ✅ Safari (iOS)
- ✅ Любой другой браузер

### Шаг 3: Введите адрес

```
http://192.168.31.227:8080
```

**Важно:**
- Используйте `http://` (не `https://`)
- Порт `:8080` обязателен
- Без пробелов

### Шаг 4: Если не работает — попробуйте другое

**Проблема:** Страница не загружается

**Решение 1: Проверьте брандмауэр**
```
Панель управления → Брандмауэр → Дополнительные параметры
→ Входящие правила → TG WS Proxy (должно быть включено)
```

**Решение 2: Временно отключите брандмауэр**
```
Панель управления → Брандмауэр → Включить/отключить
→ Отключить (только для проверки!)
```

**Решение 3: Проверьте антивирус**
- Некоторые антивирусы блокируют подключения
- Попробуйте временно отключить

## 🔍 Диагностика

### Проверка с компьютера

Откройте командную строку и проверьте:

```bash
# Проверка порта
netstat -an | findstr ":8080"
```

**Должно быть:**
```
TCP    0.0.0.0:8080           0.0.0.0:0              LISTENING
```

### Проверка с телефона

**Вариант 1: Откройте localhost на компьютере**
```
http://localhost:8080
```
Если работает — проблема с сетью.

**Вариант 2: Проверьте IP компьютера**
```
http://127.0.0.1:8080
```
Если работает — проблема с доступом из сети.

## ⚠️ Частые проблемы

### ❌ "Не удается подключиться к серверу"

**Причины:**
1. Телефон не в той сети
2. Брандмауэр блокирует
3. Антивирус блокирует
4. Неверный IP адрес

**Решение:**
1. Проверьте Wi-Fi сеть
2. Отключите брандмауэр для проверки
3. Проверьте IP: `ipconfig`
4. Перезапустите сервер

### ❌ "Превышено время ожидания"

**Причины:**
1. Сервер не запущен
2. Порт занят
3. Сетевая проблема

**Решение:**
```bash
# Перезапустите сервер
taskkill /F /FI "WINDOWTITLE eq python.exe*"
python run_web.py
```

### ❌ "404 Not Found"

**Причина:** Неверный URL

**Решение:**
```
Правильно: http://192.168.31.227:8080
Неправильно: http://192.168.31.227
             http://192.168.31.227:8080/
             https://192.168.31.227:8080
```

### ❌ Страница загружается, но не работает

**Причины:**
1. JavaScript заблокирован
2. Кэш браузера
3. Service Worker ошибка

**Решение:**
1. Очистите кэш браузера
2. Включите JavaScript
3. Используйте режим инкогнито

## 📞 Если всё ещё не работает

**Проверьте:**

1. **Брандмауэр Windows:**
   ```
   Панель управления → Брандмауэр → Журнал
   → Посмотреть заблокированные подключения
   ```

2. **Роутер:**
   - Некоторые роутеры блокируют локальные подключения
   - Попробуйте отключить "AP Isolation"

3. **Антивирус:**
   - Временно отключите
   - Добавьте в исключения

4. **Сеть:**
   ```bash
   # Проверка связи
   ping 192.168.31.227
   ```
